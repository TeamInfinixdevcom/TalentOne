# Contributing

TalentOne follows a strict foundation-first workflow.

## Working Rules

- Keep changes small and architecture-focused.
- Avoid introducing business logic, placeholder UI, or demo data in foundation work.
- Prefer shared, reusable modules over app-specific duplication.
- Keep feature boundaries explicit and domain-oriented.

## Branching And Commits

- Use short-lived feature branches.
- Write conventional commits.
- Open pull requests with clear scope, risk, and verification notes.

## Quality Bar

- TypeScript-first changes.
- Biome formatting and linting.
- Add or update documentation when repository structure changes.
